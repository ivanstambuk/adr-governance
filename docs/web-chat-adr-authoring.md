# Web-Based AI Chat ADR Authoring — Design Notes

> **Status:** Implemented  
> **Last updated:** 2026-03-06  
> **Target platforms:** ChatGPT, Microsoft Copilot (web), Claude.ai, Google Gemini

## Approach

The `adr-governance-bundle.md` (generated by `./scripts/bundle.sh`) is a single Repomix file that contains the **entire** ADR governance framework plus **embedded AI instructions** that replicate the full `adr-author` skill. Upload this one file to any web-based AI chat to get the complete ADR workflow without needing local tooling or skill execution.

## What the Bundle Contains

| Content | Source | Purpose |
|---|---|---|
| Embedded AI instructions | `repomix-instruction.md` (via `instructionFilePath`) | Tells the AI how to author, query, review, summarize, validate ADRs |
| JSON Schema | `schemas/adr.schema.json` | Ground truth for all valid fields, enums, constraints |
| Agent Skill | `.skills/adr-author/SKILL.md` + references + template | Full authoring workflow specification |
| Governance process | `docs/adr-process.md` | Status lifecycle, PR workflows, review rules |
| Glossary | `docs/glossary.md` | All term definitions, enum values |
| CI/CD setup | `docs/ci-setup.md` | Platform-specific pipeline setup |
| Existing ADRs | `architecture-decision-log/*.yaml` | The user's actual decision log |
| Example ADRs | `examples-reference/*.yaml` | Style reference (fictional NovaTrust examples) |
| Validation scripts | `scripts/*.py` | Validation logic the AI can reference |
| Governance config | `.adr-governance/config.yaml` | Governance rules |

**Bundle size:** ~350 KB (~80K tokens) — well within all platform limits.

## How `instructionFilePath` Works

Repomix supports a `"instructionFilePath"` config option that embeds a Markdown file at the end of the bundle under a `# Instruction` heading. The bundle's built-in preamble tells the AI: *"Pay special attention to the Repository Instruction."*

This replaces the need for the SKILL.md to be executed as a native skill — the instruction content covers all 7 capabilities:

1. **ADR authoring** (Socratic dialogue with step-by-step questioning)
2. **ADL querying** (search the decision log, respond with proper ADR ID citations)
3. **ADR review** (structured completeness audit with verdict)
4. **ADR summarization** (email/chat formats for stakeholders)
5. **Lifecycle transitions** (supersession, deprecation, archival)
6. **Process explanation** (lifecycle, rules, governance config)
7. **Validation** (schema compliance + semantic consistency checks)

The instruction file (`repomix-instruction.md`) also guides the AI to **use search/grep** to navigate the bundle rather than trying to ingest it all at once — critical for platforms like ChatGPT where Code Interpreter can programmatically search uploaded files.
